初めてこのプロジェクトを知るメンバーでも、「自分が何を作ればいいのか」「他の人とどう連携するのか」がパッと見てわかるように、役割と通信のルールを明確にしています。そのままコピーして共有資料としてお使いください。

# 1. 📂 プロジェクト全体概要（全員共有用）

## プロジェクトの目的と概要

本プロジェクトでは、複数人で遊べるオンラインお絵描きクイズゲーム（通称：お絵描きの森）をJavaのソケット通信を用いて制作します。

この制作の最大の裏テーマは「クライアント・サーバー間のネットワーク通信を学ぶこと」です。そのため、フロントエンドとバックエンドで分業するのではなく、機能ごとに縦割り（A・B・C）で担当を分け、全員が「画面UI」から「サーバー側でのデータ処理」まで一貫して開発を経験できる体制をとります。

## ゲームの基本ルール

1. 誰か1人が「部屋（ルーム）」を作成し、他の人がそこに参加する。
2. 全員揃ったらゲーム開始。システムがランダムに「絵を描く人」と「当てる人」に分ける。
3. 描く人には「お題（例：りんご）」が提示され、キャンバスに絵を描く。
4. 当てる人はチャットで回答を送信。早く正解した人ほど高得点。
5. 制限時間が来るか、全員が正解したらその回は終了。
6. 全員が1回ずつ「描く人」を担当したら1ラウンド終了（任意ラウンド繰り返し）。
7. 最終的なスコアを表示して終了。

## 開発体制と役割分担

衝突（コンフリクト）を防ぐため、以下の4つの役割に完全に分割して開発を進めます。

* **担当A：ルーム管理**（部屋の作成・参加・待機）
* **担当B：お絵描き**（マウス座標の取得・描画・リアルタイム同期）
* **担当C：ゲーム進行・チャット**（お題の管理・正誤判定・スコア計算）
* **担当D（基盤）：通信土管の作成**（サーバー・クライアント間の接続維持とデータ振り分け）※作成済み

## 開発の絶対ルール：通信コマンド（Protocol）

全員が1つの通信回線（ソケット）を共有するため、勝手にデータを送るとバグの原因になります。
データを送受信する際は、必ず `common/Protocol.java` に「コマンド名」を定義し、「コマンド:データ内容」という形式（例：`CHAT_SEND:りんご`）で通信を行ってください。新しい通信が必要になったら、まずはチームで合意をとってProtocolに追加しましょう。

これまでのやり取りと、共有していただいた画像に基づいた**最新のディレクトリ構造**と、**各ファイルが具体的に何をしているのかの解説**をまとめました。

このリストが、チーム全員にとっての「全体マップ」になります。

### 📄 各ファイルの役割解説

#### 【共通領域】全員が参照するルールブック

* **`common/Protocol.java`**
* **役割：** 通信で使う「合言葉（コマンド）」を定数として定義するファイルです。
* **なにしてるか：** 例えば「線を引いたよ」というデータを送るときに、`"DRAW"` という文字列を使うのか `"LINE"` を使うのか、チーム内で表記揺れが起きないようにここで `public static final String DRAW_LINE = "DRAW_LINE";` のように定義し、全員がこれを参照します。



#### 【サーバー側】基盤担当（D）の領域

* **`server/GameServer.java`**
* **役割：** サーバーの「受付係」です。
* **なにしてるか：** ポート番号（8080など）を指定して起動し、新しいプレイヤー（クライアント）が接続してくるのをずっと待ち構えます。誰かが接続してきたら、その人専用の `ClientHandler` を新しく作って対応を任せます。


* **`server/ClientHandler.java`**
* **役割：** 各プレイヤー専属の「電話交換手」です。
* **なにしてるか：** 繋がったプレイヤーから送られてくる文字列（例：`ROOM_CREATE:ライオン部屋`）を常時読み取ります。そして、`Protocol.java` の合言葉をもとに「これは部屋作成の依頼だから Aさん（RoomManager）に渡そう」「これはお絵描きデータだから Bさん（DrawManager）に渡そう」と、**`switch` 文を使って各Managerへ処理を丸投げ（振り分け）します。**



#### 【サーバー側】A・B・Cの領域（処理の実行部）

* **`server/room/RoomManager.java` (担当A)**
* **役割：** 部屋の管理者です。
* **なにしてるか：** `ClientHandler` から「部屋を作って」「部屋に入りたい」という依頼を受け取り、現在ある部屋のリストや、誰がどの部屋にいるかという情報をサーバーのメモリ上で管理・更新します。


* **`server/draw/DrawManager.java` (担当B)**
* **役割：** お絵描きデータの中継地点です。
* **なにしてるか：** 誰かが絵を描いたときの座標データを `ClientHandler` から受け取り、同じ部屋にいる「他の全員の `ClientHandler`」を探して、その座標データをそのまま一斉送信（ブロードキャスト）します。


* **`server/game/GameManager.java` (担当C)**
* **役割：** ゲームの審判であり、進行役です。
* **なにしてるか：** お題を選んで各プレイヤーに配ったり、送られてきたチャット内容がお題と一致しているか（正解か）を判定したりします。また、制限時間やスコアの計算もここで行います。



#### 【クライアント側】基盤担当（D）の領域

* **`client/GameClient.java`**
* **役割：** プレイヤーのPC（クライアント）側の通信窓口です。
* **なにしてるか：** サーバーのIPアドレスとポートを指定して接続（Socket通信）を行います。また、「いつサーバーからデータが送られてきてもいいように、常に耳をすませておく裏方のプログラム（受信スレッド）」を走らせ、データが届いたらクライアント側の各Controllerへ振り分けます。



#### 【クライアント側】A・B・Cの領域（各自がこれから作るファイル）

* **`client/room/` (担当A)**
* **なにしてるか：** プレイヤーが部屋名を入力する画面（UI）や、その入力内容を `ROOM_CREATE` コマンドとしてサーバーに送信する処理（Controller）をここに作っていきます。


* **`client/draw/` (担当B)**
* **なにしてるか：** マウスで線を描けるキャンバス画面（UI）と、マウスが動いた座標を取得してサーバーに送信したり、逆にサーバーから送られてきた他人の線を描画したりする処理（Controller）をここに作っていきます。


* **`client/game/` (担当C)**
* **なにしてるか：** チャットを入力する画面や残り時間を表示する画面（UI）と、入力したチャット内容をサーバーに送信したり、サーバーから「正解！」という結果を受け取って画面にスコアを表示したりする処理（Controller）をここに作っていきます。

# 2. 📝 担当A：ルーム管理チーム 仕様書

**ミッション：**
プレイヤーがゲームを始めるための「ロビー（待合室）」を作ることです。誰がどの部屋にいるのかを管理し、BさんやCさんがゲームを始められる土台を整えます。

## 実装する機能

1. **部屋の作成**：プレイヤーが任意の部屋名を入力して部屋を作れるようにする。
2. **部屋への参加**：すでに作成されている部屋の一覧を表示し、選んで参加できるようにする。
3. **待機画面の表示**：現在同じ部屋に誰がいるのかリスト表示し、「ゲーム開始」ボタンを配置する（開始処理自体はCが担当）。

## 編集する主なファイル

* `client/room/RoomPanel.java` (画面：部屋作成・一覧表示UI)
* `client/room/RoomController.java` (送信：サーバーへのリクエスト送信)
* `server/room/RoomManager.java` (受信：サーバー側での部屋リスト・メンバー管理)

## 通信のイメージ

* **送信**：`ROOM_CREATE:ライオン部屋` （部屋「ライオン部屋」を作って！）
* **受信**：`ROOM_CREATED_NOTIFY:ライオン部屋,成功` （サーバーからの作成完了通知）
* **送信**：`ROOM_JOIN:ライオン部屋,ユーザー名` （その部屋に入れて！）

## 開発のヒント

まずは見た目（UI）をシンプルに作り、ボタンを押したときに `System.out.println` で正しく文字列が生成されるか確認するところから始めるとスムーズです。

# 3. 🎨 担当B：お絵描きチーム 仕様書

**ミッション：**
このゲームの心臓部である「キャンバス」を作ることです。一人が描いたマウスの軌跡をデータに変換し、他の全員の画面にリアルタイムで魔法のように表示させます。

## 実装する機能

1. **お絵描きキャンバス**：マウスのドラッグ操作を検知し、画面に線を引く。
2. **色の変更・消しゴム**：ペンの色を変えたり、キャンバスをクリアしたりする機能。
3. **座標データの送受信**：「どこからどこまで線を引いたか」の座標データをサーバーに送り、他人が描いた座標データを受け取って自分の画面にも描画する。

## 編集する主なファイル

* `client/draw/DrawPanel.java` (画面：キャンバスUI、マウスイベント取得)
* `client/draw/DrawController.java` (送信：座標データの文字列化と送信)
* `server/draw/DrawManager.java` (受信：受け取った座標を同じ部屋の他メンバーに転送)

## 通信のイメージ

* **送信**：`DRAW_LINE:150,200,152,205,BLACK` （X1,Y1からX2,Y2へ黒い線を引いたよ！）
* **受信**：`DRAW_LINE:150,200,152,205,BLACK` （サーバー経由で他の人の線データが届く）

## 開発のヒント

マウスが1ミリ動くたびに通信するとデータがパンクします。最初はカクカクでもいいので、座標を文字列（カンマ区切りなど）に変換して送る・受け取って元に戻す（シリアライズ・デシリアライズ）処理に集中しましょう。

# 4. 🎮 担当C：ゲーム進行・チャットチーム 仕様書

**ミッション：**
ゲームの「ルールと審判」を作ることです。いつ始まり、誰が絵を描き、誰が正解し、いつ終わるのかという状態（ステート）を管理し、プレイヤーに熱狂をもたらします。

## 実装する機能

1. **チャット機能**：プレイヤーがテキストを入力して送信し、全員の画面に表示させる。
2. **お題の選出と役割振り分け**：「描く人」にはお題をこっそり教え、「当てる人」には隠す。
3. **正誤判定とスコア管理**：送られてきたチャットがお題と完全一致するかチェック。正解ならスコアを加算し、制限時間やターンの管理を行う。

## 編集する主なファイル

* `client/game/ChatPanel.java` (画面：チャット入力欄、ログ表示、スコア・残り時間表示)
* `client/game/GameController.java` (送信：チャット発言の送信)
* `server/game/GameManager.java` (受信：正誤判定、タイマー処理、全員への結果通知)

## 通信のイメージ

* **サーバーから**：`GAME_ROUND_START:DRAWER,りんご` （あなたは描く人！お題はりんご！）
* **送信**：`CHAT_SEND:みかん` （みかんだと思う！）
* **サーバーから**：`GAME_JUDGE:USER1,不正解` （USER1の回答はハズレです）
* **サーバーから**：`SCORE_UPDATE:USER1=50,USER2=10` （現在の得点状況）

## 開発のヒント

一番ロジックが複雑な担当です。まずは純粋な「チャットアプリ（入力した文字が全員に届く）」を作ることを第一目標にしてください。それができたら、サーバー側で「入力された文字 == お題」を判定するIF文を追加していくと分かりやすいです。
